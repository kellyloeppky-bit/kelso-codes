### Q1. What should you create before writing any code for a new project?

- A. A Git repository
- B. A functional design document with data models
- C. Unit tests
- D. A marketing website

**Correct answer: B** — Drawing your data model and mapping entity relationships before writing code helps you understand whether you actually grasp the problem. Skip this step and you'll likely rebuild the same thing multiple times.

---

### Q2. When should security features like MFA and role-based access control be implemented?

- A. After the MVP is complete
- B. When you get your first user
- C. From day one
- D. Only if you're handling sensitive data

**Correct answer: C** — Identity management, MFA, and role-based access control aren't features you add later. They're foundational decisions that shape your entire architecture. By the time you're thinking about security at the end, your system is already working against you.

### Q3. What is the main advantage of using multiple AI models together?

- A. It costs less than using one premium model
- B. Each model has different strengths for different tasks
- C. It makes your code more complex and impressive
- D. You can compare outputs and pick the best one

**Correct answer: B** — The real power isn't any one tool, but knowing how to connect them. Use Perplexity for real-time research, Claude for reasoning and strategy, and Gemini when you need a different perspective on a hard problem.

### Q4. What does "separation of concerns" mean in software development?

- A. Keeping work and personal projects separate
- B. Distributing code to where it logically belongs instead of one large file
- C. Separating frontend from backend
- D. Having different developers work on different features

**Correct answer: B** — Pulling code out of a massive main file and distributing it to where it actually belongs is called separation of concerns. You often can't see the mess until you've lived in it long enough.

---

### Q5. What is a structured instruction file for AI?

- A. A configuration file that sets API parameters
- B. A document that tells an AI model exactly how to behave for a specific task
- C. A training dataset in structured format
- D. A SQL schema file

**Correct answer: B** — Think of structured instruction files as a recipe the AI follows before it starts cooking. The words matter, the order matters, and what you leave out matters as much as what you put in.

---

### Q6. Why is choosing a framework considered a "long-term relationship"?

- A. Frameworks receive updates for many years
- B. You'll need to pay licensing fees annually
- C. Changing it later isn't refactoring, it's starting over
- D. Framework communities are very supportive

**Correct answer: C** — Whatever framework you choose, understand what you're committing to. Flexibility early has a cost as complexity grows. Changing it later isn't refactoring. It's starting over.

---

### Q7. What is data contract design in distributed systems?

- A. Legal agreements about who owns the data
- B. How services pass information to each other - what goes in, what comes back
- C. Database schema versioning
- D. API rate limiting policies

**Correct answer: B** — How your services pass information to each other, what goes in, what comes back, what gets stored, determines whether your system is reliable or fragile. Get the contract right.

---

### Q8. What happens when you build something for someone else versus for yourself?

- A. It takes twice as long
- B. You need more budget
- C. You learn faster because the stakes are real
- D. You need a formal project plan

**Correct answer: C** — There is a specific kind of focus that arrives when you know someone else is going to click your buttons in front of their boss. Ship for yourself, learn for yourself. Build for someone else, learn faster.

---

### Q9. What is the lesson about refactoring code?

- A. Always refactor before adding new features
- B. You can't see the mess until you've lived in it long enough
- C. Use automated refactoring tools
- D. Refactor every Friday

**Correct answer: B** — The first version worked. The second version was cleaner. The third version was honest. The lesson isn't the refactor. The lesson is that you can't see the mess until you've lived in it long enough.

### Q10. What tool can turn documents into audio intelligence?

- A. Claude
- B. Perplexity
- C. Google NotebookLM
- D. Gemini

**Correct answer: C** — Google NotebookLM turns documents into audio intelligence, providing another way to consume and understand information from your documents.
