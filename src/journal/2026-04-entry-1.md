### Q1. What was the root cause of the Auth0 login failure after migrating Gossip Island to its custom domain?

- A. The Auth0 tenant was deleted during migration
- B. A hardcoded SESSION_TYPE = 'null' was overriding the config, and Auth0 callback URLs hadn't been updated
- C. The SSL certificate on Directnic expired
- D. The Flask secret key was rotated during deployment

\*\*Correct answer: B\*\* — Two things needed fixing: the Auth0 allowed callback URLs had to be updated to https://www.gossipisland.com, and a hardcoded SESSION_TYPE = 'null' in root app.py was overriding the filesystem session config.

### Q2. What accuracy protocol did you develop and apply to your research prompts?

- A. A chain-of-thought prompt that forces the model to cite Wikipedia before answering
- B. A protocol requiring knowledge boundary declarations, no fabrication, recency flagging, source transparency, and a confidence signal at the end
- C. A simple instruction to only use government websites as sources
- D. A template that asks the model to write in APA format and nothing else

\*\*Correct answer: B\*\* — The accuracy protocol enforces knowledge boundary declarations, prohibits fabrication, requires recency flagging, demands source transparency, and ends each response with a confidence signal. Works across Claude, ChatGPT, and Gemini.

### Q3. What DNS record type did you use to point www.gossipisland.com to your Render deployment on Directnic?

- A. A (address) record pointing to Render's IP
- B. CNAME record pointing to gossip-island.onrender.com
- C. MX record with the Render hostname
- D. TXT record with the site verification token

\*\*Correct answer: B\*\* — A CNAME record for 'www' pointing to gossip-island.onrender.com was used. An A record (216.24.57.1) handled the bare domain.

### Q7. What error did you encounter when setting up the Windows MCP server in Claude Desktop, and what caused it?

- A. 'spawn uv ENOENT' — the uv Python package manager wasn't installed or in the system PATH
- B. 'CORS blocked' — Claude Desktop doesn't allow local HTTP connections
- C. 'Module not found: windows-mcp' — the package wasn't installed via npm
- D. 'Permission denied' — the MCP server requires admin privileges to run

\*\*Correct answer: A\*\* — The error 'spawn uv ENOENT' meant Claude Desktop couldn't find the uv executable. Fix: install uv via PowerShell and restart Claude Desktop.
